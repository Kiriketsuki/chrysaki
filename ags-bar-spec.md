# AGS Bar — Feature Specification

**Status:** Draft
**Phase:** Chrysaki Design System — Phase 3
**Author:** kiriketsuki
**Date:** 2026-03-15

---

## 1. Overview

Replace Waybar with a fully custom AGS bar that expresses the Chrysaki design language: zero border-radius, trapezoid tiling, jewel-tone accents, and glassmorphism blur. The bar is a single floating island anchored centre-top of the primary monitor.

### Goals

- Full Chrysaki design compliance (zero border-radius, zigzag-alt tiling, OKLCH palette)
- Live Hyprland workspace state via native IPC
- Service health indicators for the personal stack (OV, GitNexus, Ollama, Syncthing)
- Stow-compatible: `cd ~/dots/chrysaki && stow ags` installs the bar

### Non-Goals

- Waybar replacement on secondary monitors (out of scope for v1)
- Multi-theme switching (Chrysaki palette is fixed)
- Vertical bar orientation

## 3. Layout

### Floating Island (centre-top)

```
┌──────────────────────────────────────────────────────────┐
│ [ws-pips] [clock] [media] [sys-tray] [net] [bat] [power] │
└──────────────────────────────────────────────────────────┘
  emerald    teal   amethyst  secondary  blue  blonde  ruby
```

- Anchored: `top-centre` via gtk-layer-shell
- Margins: 8px top, horizontally centred
- Height: 36px
- Width: content-fit with min/max constraints

### Tiling

The island uses the `zigzag-alt` preset (`first_left=S, first_right=\`) from `TILING.md`. Each widget group is one tiling segment. CSS renderer rules per `TILING.md §8.1`:

- Inset: **16px**
- Overlap: `margin-left: -16px` on all segments except first
- Element shapes generated by porting `tools/tiling.py` `to_css_clip_paths()` to TypeScript, or calling it at build time to emit SCSS `%clip-*` placeholder classes

**Segment sequence (7 segments, `zigzag-alt` N=7):**

| # | Widget | Edges | clip-path |
|:--|:-------|:------|:----------|
| 1 | WorkspaceIndicator | `(S, \)` | `polygon(0% 0%, calc(100% - 16px) 0%, 100% 100%, 0% 100%)` |
| 2 | Clock | `(\, /)` | `polygon(0% 0%, 100% 0%, calc(100% - 16px) 100%, 16px 100%)` |
| 3 | MediaPlayer | `(/, \)` | `polygon(16px 0%, calc(100% - 16px) 0%, 100% 100%, 0% 100%)` |
| 4 | SystemTray | `(\, /)` | `polygon(0% 0%, 100% 0%, calc(100% - 16px) 100%, 16px 100%)` |
| 5 | Network | `(/, \)` | `polygon(16px 0%, calc(100% - 16px) 0%, 100% 100%, 0% 100%)` |
| 6 | Battery | `(\, /)` | `polygon(0% 0%, 100% 0%, calc(100% - 16px) 100%, 16px 100%)` |
| 7 | PowerMenu | `(/, S)` | `polygon(16px 0%, 100% 0%, 100% 100%, 0% 100%)` |

*Service status indicators (OV/GitNexus/Ollama/Syncthing) sit inside the SystemTray segment or as micro-indicators on a dedicated segment (TBD during implementation).*

## 4. Widget Specifications

### 4.1 WorkspaceIndicator — Must-Have

**Domain:** Emerald
**Data source:** `libastal-hyprland` Hyprland IPC

Renders one pip per workspace. Pip size: 8×8px (hexagon flat-top shape via `clip-path`).

| State | Color |
|:------|:------|
| Active | Cycles: Emerald Lt → Blue Lt → Amethyst Lt (by workspace index mod 3) |
| Occupied (not active) | Dim variant of the cycling color |
| Urgent | `$error` (`#8C2F39`) + pulse animation (Nice-to-Have) |
| Inactive (empty) | `$border` (`#363a4f`) |

Interaction: clicking a pip dispatches `hyprctl dispatch workspace N`.

### 4.2 Clock — Must-Have

**Domain:** Teal
**Data source:** GLib `GLib.DateTime` polling, 1s interval

Format: `HH:MM` (24h). Font: IosevkaTermSlab, weight 600.
Color: `$teal-light` (`#20969c`).
Seconds may be added as `$text-muted` subscript (Nice-to-Have).

### 4.3 Battery — Must-Have

**Domain:** Blonde → Error
**Data source:** `libastal-battery` (UPower)

- Full / charging: `$blonde` (`#FBB13C`)
- Medium (20–80%): `$blonde-dim` (`#C4861C`)
- Low (<20%): `$error` (`#8C2F39`)
- Critical (<5%): `$error` + pulse animation (Nice-to-Have)

Displays percentage and a Unicode battery icon. Hides when no battery device present.

### 4.4 PowerMenu button — Must-Have

**Domain:** Error (Ruby)
**Data source:** none (button)

Single icon button. Click spawns `rofi -show power-menu -modi "power-menu:rofi-power-menu"` (or `hypr/scripts/powermenu.sh`).
Color: `$error` (`#8C2F39`). Icon: `⏻`.

### 4.5 Glassmorphism background — Must-Have

Applied to the top-level island container:

```scss
background: rgba(255, 255, 255, 0.05);
// backdrop-filter applied via Hyprland layerrule, NOT CSS
border: 1px solid rgba(255, 255, 255, 0.08);
box-shadow: 0 4px 30px rgba(0, 0, 0, 0.30);
```

Hyprland `hyprland.conf` additions:
```
layerrule = blur, gtk-layer-shell
layerrule = ignorezero, gtk-layer-shell
```

> **Open Question #1**: GTK4 `clip-path` + Hyprland layerrule blur interaction. The Firefox fix (wrapper carries `clip-path`, child carries `backdrop-filter`) may be required. If blur does not respect `clip-path` on the island, a wrapper+child split is the fallback — outer `<box>` owns the shape, inner `<box>` carries the glass tint.

## 5. Should-Have Widgets

### 5.1 SystemTray

**Domain:** Secondary (neutral)
**Data source:** `libastal-tray`

Renders XDG tray icons. No custom styling per icon; icons use their native pixbuf. Segment background: `$surface`.

### 5.2 MediaPlayer

**Domain:** Amethyst
**Data source:** `libastal-mpris`

Displays: artist, track title (truncated at 24 chars), play/pause state icon.
Colors: `$amethyst-light` text on `$amethyst-dim` background.

> **Open Question #2**: Does `libastal-mpris` expose album art (as a file path or pixbuf), or only text metadata (artist/title/status)? If album art is available, display as a 24×24px hexagon thumbnail left of the title.

### 5.3 Network

**Domain:** Blue
**Data source:** `libastal-network` (NetworkManager)

Displays: WiFi SSID (truncated at 16 chars) + signal strength icon, or Ethernet icon.
Color: `$blue-light` (`#1c3d7a`). Uses fill color only (not as text) per CLAUDE.md design rules — icon is `$teal-light`, label is `$text-secondary`.

### 5.4 Service Status Indicators

**Domain:** Mixed (per service)
**Data source:** `Variable("").poll(interval, ["bash", "-c", "command"])`

Four micro-indicators displayed as 6×6px hexagon dots inside or adjacent to the SystemTray segment:

| Service | Icon | Poll | Healthy | Unhealthy |
|:--------|:-----|:-----|:--------|:----------|
| OV | `⬡` | 5s | `$emerald-light` | `$error` |
| GitNexus | `⬡` | 10s | `$teal-light` | `$error` |
| Ollama | `⬡` | 10s | `$amethyst-light` | `$error` |
| Syncthing | `⬡` | 15s | `$blonde` | `$error` |

Poll commands:

```bash
# OV — poll health endpoint (URL TBD)
# Open Question #3: what URL/socket does OV expose for health checks?
curl -sf http://localhost:PORT/health && echo ok || echo fail

# GitNexus — check if npx gitnexus is runnable (process or binary check)
command -v npx && npx gitnexus status --quiet && echo ok || echo fail

# Ollama
curl -sf http://localhost:11434/api/version && echo ok || echo fail

# Syncthing
systemctl --user is-active syncthing && echo ok || echo fail
```

## 9. Exit Criteria

- [ ] All Must-Have widgets render on an active Hyprland session

- [ ] Workspace pips reflect live Hyprland workspace state (active/occupied/urgent/inactive)

- [ ] No regressions to existing Chrysaki ports (tmux, nvim, vscode, etc.)
