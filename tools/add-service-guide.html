Adding Services to the Centre

The service centre is fully config-driven — no TypeScript changes needed. Drop an icon, add a JSON entry, restart AGS.

services.json · No compile step · Restart to apply
WORKFLOW · 3 STEPS
ADD ICON

Drop the SVG

Place a symbolic SVG in the icons directory. Name it <name>-symbolic.svg — the -symbolic suffix is required for GTK theming.

ags/.config/ags/icons/hicolor/scalable/status/
└── myservice-symbolic.svg
EDIT JSON

Add the entry

Append a service object to services.json. The file is stow-symlinked to ~/.config/ags/services.json — edit either path.

chrysaki/ags/.config/ags/services.json
RESTART

Kill & relaunch AGS

Services load at startup only. After any icon or JSON change, restart the process. New icons also need a stow re-run if you added the file fresh.

stow -d ~/dots/chrysaki -t ~ ags -v

kill $(pgrep -f "gjs.*ags.js")
WAYLAND_DISPLAY=wayland-1 ags run \
~/.config/ags/app.ts &
STOW NOTE  Only re-run stow when you add a new file to the package. Editing an existing file (services.json already symlinked) takes effect immediately — no stow needed.
FIELD REFERENCE · services.json
Field Type Description & accepted values
name string required Display name shown in the panel row and used as the tooltip on the bar icon. Also the key in service-prefs.json — must be unique.
type enum required "daemon" — systemd unit or HTTP service
"process" — background process (pgrep-checkable)
"cli" — installed CLI tool (version-check)
category string required Groups rows in the panel under a header. Use an existing category or introduce a new one — the icon for new categories defaults to a circle unless added to CATEGORY_ICONS in ServiceStatus.tsx.
icon string required Filename without the -symbolic.svg suffix. Falls back to application-x-executable-symbolic if the file is missing. Example: "myservice-symbolic" → use "myservice".
healthCmd string required Bash command that outputs ok or fail to stdout. Must be self-contained — runs via bash -c. See health patterns below.
startCmd string | null required Command to start the service when it's unhealthy. Set null to disable the toggle button in the unhealthy state.
stopCmd string | null required Command to stop the service when it's healthy. Set null to disable the toggle button in the healthy state.
guiCmd string | null required Command to open the service's web UI or desktop app. Renders a link button (⧉) in the row. Set null to hide — an invisible placeholder preserves column alignment.
cssAccent string required Row background tint class. See accent palette below.
interval number required Health check poll interval in milliseconds. Recommended: 5000 for HTTP, 10000 for process, 15000–30000 for slow checks.
HEALTH COMMAND PATTERNS · by type
DAEMON

HTTP health endpoint

Curl the service's health URL. Suppress all output — only the echo matters.

curl -sf http://localhost:PORT/health \
>/dev/null 2>&1 \
&& echo ok || echo fail
DAEMON

systemd unit

Use is-active for user units, drop --user for system units.

systemctl --user is-active SERVICE \
>/dev/null 2>&1 \
&& echo ok || echo fail
PROCESS

pgrep match

Match on a unique fragment of the process command line.

pgrep -f 'PROCESS_PATTERN' \
>/dev/null 2>&1 \
&& echo ok || echo fail
CLI

Version check

Any command that exits 0 when the tool is installed. Most tools accept --version.

TOOL --version \
>/dev/null 2>&1 \
&& echo ok || echo fail

CATEGORIES · existing + adding new
SYSTEM
System
Daemons and background infrastructure. Use "category": "System".
MCP
MCP
Model Context Protocol servers. Use "category": "MCP".
CLI TOOLS
CLI Tools
Installed binaries with no daemon. Use "category": "CLI Tools".
NEW CATEGORY  Use any string for category. A new category header renders automatically with a default bullet icon. To assign a Nerd Font glyph, add one entry to CATEGORY_ICONS in ServiceStatus.tsx:

"My Category": "\uf???", // nf-fa-something

That's the only TypeScript change ever needed.
COMPLETE EXAMPLE · new HTTP daemon
services.json — append to array JSON 
{
  "name":      "MyService",
  "type":      "daemon",
  "category": "System",
  "icon":      "myservice-symbolic",  // → icons/.../myservice-symbolic.svg

  "healthCmd": "curl -sf http://localhost:8080/health >/dev/null 2>&1 && echo ok || echo fail",
  "startCmd":  "systemctl --user start myservice",
  "stopCmd":   "systemctl --user stop myservice",
  "guiCmd":    "xdg-open http://localhost:8080",  // or null

  "cssAccent": "svc-emerald",
  "interval":  5000
}
