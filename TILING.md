# Chrysaki Tiling System ‚Äî Formal Specification

Renderer-agnostic rules for generating trapezoid tiling strips used across Chrysaki ports.

---

## 1. Overview

A **tiling strip** is a sequence of N trapezoid elements arranged edge-to-edge (or with pixel overlap) along a single axis. Each element has a **left edge** and a **right edge** that define its silhouette. Adjacent elements share an edge type: the right edge of element i equals the left edge of element i+1, creating seamless interlocking tessellation.

This spec is renderer-agnostic. Renderers map the abstract edge types to CSS `clip-path` polygons, terminal Nerd Font glyphs, or SVG paths.

---

## 2. Terminology

| Term | Definition |
|:-----|:-----------|
| **Element** | A single trapezoid segment in the strip, described by `(left_edge, right_edge)` |
| **Edge type** | One of three values: `S` (straight / pipe), `/` (forward slash), `\` (backslash) |
| **Tiling strip** | An ordered sequence of N elements generated by the tiling algorithm |
| **Inset** | The pixel depth of a diagonal cut (default 16px for CSS, glyph-native for terminal) |
| **Overlap** | How many pixels adjacent elements overlap to hide the gap between them (equals inset for CSS) |
| **first_left** | The left edge of the first element ‚Äî caller-specified |
| **first_right** | The right edge of the first element ‚Äî caller-specified |

*Note: "straight" edge is written `S` in prose and tables (to avoid breaking markdown table formatting with `|`), but is represented as the `|` character in code, CLI arguments, and ASCII diagrams.*

---

## 3. Edge Algebra

```
opposite(/)  = \
opposite(\)  = /
opposite(S)  = S     (straight is self-opposite)
```

The straight `S` edge terminates the strip rather than continuing the zigzag oscillation.

---

## 4. Tiling Algorithm

```
tile(first_left, first_right, N):

  if N = 1:
    Element 1: (first_left, S)          # force termination on both sides

  if N > 1:
    Element 1:   (first_left, first_right)
    Element i    (1 < i < N):   (prev.right, opposite(prev.right))
    Element N    (last, N > 1): (prev.right, S)
```

Rules:
- The last element always terminates with `S` (straight right edge).
- Middle elements alternate by applying `opposite()` to the previous element's right edge.
- Only `first_left` and `first_right` are free parameters; everything else is derived.

---

## 5. RTL Mirror

To render a strip right-to-left (e.g., reversed tab bars, right-anchored status segments):

1. Reverse the element order.
2. In each (now-reversed) element, swap left and right edges, then flip `/` ‚Üî `\` on both.

```
mirror(elements):
  return [Element(flip(el.right), flip(el.left)) for el in reversed(elements)]

flip(/)  = \
flip(\)  = /
flip(S)  = S
```

## 8. Renderer Mapping

### 8.1 CSS `clip-path: polygon()`

Each element is a rectangle `W √ó H` with pixel inset `I`. Points listed clockwise:
top-left ‚Üí top-right ‚Üí bottom-right ‚Üí bottom-left.

**Left edge ‚Üí top-left (TL) and bottom-left (BL):**

| Edge | TL | BL |
|:-----|:---|:---|
| S (straight) | `0px 0%` | `0px 100%` |
| `/` | `Ipx 0%` | `0px 100%` |
| `\` | `0px 0%` | `Ipx 100%` |

**Right edge ‚Üí top-right (TR) and bottom-right (BR):**

| Edge | TR | BR |
|:-----|:---|:---|
| S (straight) | `100% 0%` | `100% 100%` |
| `/` | `100% 0%` | `calc(100% - Ipx) 100%` |
| `\` | `calc(100% - Ipx) 0%` | `100% 100%` |

**Full formula:**
```
clip-path: polygon(TL, TR, BR, BL);
```

**Overlap / margin:** set `margin-left: -Ipx` on all elements except the first to close the gap. Default I = 16px.

**Example ‚Äî element `(/, \)`:**
```css
clip-path: polygon(16px 0%, calc(100% - 16px) 0%, 100% 100%, 0% 100%);
margin-left: -16px;
```

### 8.2 Terminal ‚Äî Nerd Font Glyphs

| Edge | Unicode | Visual |
|:-----|:--------|:-------|
| `/` | U+E0B8 | Forward-slash diagonal separator |
| `\` | U+E0BC | Backslash diagonal separator |
| S (straight) | U+E0B0 | Straight solid-fill cap |

The glyph character sits between two tmux color segments. The glyph foreground = background of the segment being left; glyph background = background of the segment being entered. This creates the filled separator effect.

### 8.3 SVG

Same geometry as CSS (¬ß8.1) but with absolute coordinates. For element width W, height H, inset I:

| Edge | Points |
|:-----|:-------|
| S left | TL: (0, 0)     BL: (0, H) |
| `/` left | TL: (I, 0)     BL: (0, H) |
| `\` left | TL: (0, 0)     BL: (I, H) |
| S right | TR: (W, 0)     BR: (W, H) |
| `/` right | TR: (W, 0)     BR: (W‚àíI, H) |
| `\` right | TR: (W‚àíI, 0)   BR: (W, H) |

```xml
<!-- Example: element (/, \) at W=120, H=32, I=16 -->
<polygon points="16,0 104,0 120,32 0,32" />
```

## 10. Overlap Defaults

| Renderer | Default inset | Default overlap mechanism |
|:---------|:-------------|:--------------------------|
| CSS | 16px | `margin-left: -16px` on all elements except first |
| Terminal | Glyph-native | Nerd Font glyphs are single-cell; no margin adjustment needed |
| SVG | 16px | Translate each element left by 16px except first; z-order: later = on top |
